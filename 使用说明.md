## 快速开始

### 安装依赖

```bash
# 后端
cd backend
pip install -r requirements.txt

# 前端
cd frontend
npm install
```

### 启动服务

```bash
# 后端（终端 1）
cd backend
python -m uvicorn main:app --reload --port 8000

# 前端（终端 2）
cd frontend
npm run dev
```

访问 http://localhost:5173 即可使用。

### 配置 API

首次使用需要在 **设置** 页面配置 AI 模型连接：

| 模型 | 配置项 |
|------|--------|
| Ollama | `OLLAMA_BASE_URL`（默认 http://localhost:11434） |
| DeepSeek | `DEEPSEEK_API_KEY` + `DEEPSEEK_BASE_URL` |
| OpenAI | `OPENAI_API_KEY` + `OPENAI_BASE_URL` |
| Claude | `CLAUDE_API_KEY` + `CLAUDE_BASE_URL` |
| Azure | `AZURE_API_KEY` + `AZURE_ENDPOINT` |

## 功能概览

AstralLounge 基于 SillyTavern 设计理念，是一款功能强大的 AI 对话前端，支持：

- 多模型 AI 对话（Ollama / DeepSeek / OpenAI / Claude / Azure）
- 角色卡管理与导入
- 多角色群聊
- 世界设定（Lorebook）动态注入
- 记忆系统（自动 + 手动）
- 用户人设（Personas）
- Prompt 管理器（可视化编辑提示词组件）
- 图片生成（Stable Diffusion / DALL-E）
- 语音合成（TTS）
- 插件系统（auto-memory、web-search）
- 通知中心
- 深色/浅色主题 + 移动端适配

## 角色管理

### 创建角色

1. 点击左侧导航 **角色** 按钮
2. 点击 **创建角色** 按钮
3. 填写角色信息：
   - **名称**：角色名字
   - **描述**：角色外观和背景
   - **性格**：角色性格特点
   - **场景**：对话场景设定
   - **开场白**：首次对话时角色说的话
   - **示例对话**：对话示例（可提升角色扮演质量）
   - **头像**：上传角色图片

### 角色卡导入

支持导入 SillyTavern / TavernAI 格式的角色卡（JSON / PNG）。

### 标签管理

为角色添加标签，便于分类和筛选。

## 对话聊天

### 开始对话

1. 从角色列表选择一个角色
2. 角色卡片下方显示 **新对话** 按钮
3. 点击开始新的对话会话

### 发送消息

- 在底部输入框输入内容
- 按 **Enter** 或点击发送按钮
- AI 以流式方式逐字输出（打字机效果）

### 对话设置

每个会话可以单独设置：

- **模型选择**：覆盖全局默认模型
- **Temperature**：控制回复随机性（0.1 - 2.0）
- **Max Tokens**：单次回复最大 token 数
- **Token 计数**：实时显示上下文 token 使用量

### 上下文 Token 警告

当 token 使用超过上下文的 80% 时，显示警告提示，支持一键摘要压缩上下文。

## 群聊功能

### 创建群聊

1. 点击 **群聊** 页面
2. 点击 **创建群聊** 按钮
3. 输入群聊名称
4. 从角色库中选择成员（至少 2 个）

### 群聊回复策略

| 策略 | 说明 |
|------|------|
| **手动** | 用户选择发言角色 |
| **自然顺序** | 根据发言频率和提及自动决定 |
| **列表顺序** | 按成员列表顺序轮流发言 |
| **池化顺序** | 随机选择未发言的成员 |

### 群聊设置

- **成员管理**：添加、移除、禁言成员
- **场景覆盖**：所有成员使用统一的场景设定
- **自动模式**：开启后自动连续生成回复

## 世界设定 (Lorebook)

世界设定是 SillyTavern 最强大的功能之一，允许在对话中动态插入背景信息。

### 核心概念

世界设定由多个 **条目（Entry）** 组成，每个条目包含：

| 字段 | 说明 |
|------|------|
| **名称** | 条目标题（仅供识别，不参与匹配） |
| **内容** | 实际插入到提示词中的文本 |
| **关键词** | 触发该条目的关键词列表 |
| **优先级** | 数值越大越靠后插入 |
| **触发策略** | constant / keyword / chain |

### 触发策略

| 策略 | 图标 | 说明 |
|------|------|------|
| **常驻** | 🔵 | 始终插入，不依赖关键词 |
| **关键词** | 🟢 | 仅当消息包含关键词时插入 |
| **链式** | 🔗 | 可被其他条目递归触发 |

### 触发概率

每个条目可以设置 0-100% 的触发概率。设置为 50% 表示只有一半的几率被插入。可用于创建随机事件。

### 排除条件

可选设置排除逻辑，在满足某些条件时不触发条目：

| 逻辑 | 说明 |
|------|------|
| AND ANY | 包含任一排除关键词则不插入 |
| AND ALL | 包含全部排除关键词才不插入 |
| NOT ANY | 不包含任一排除关键词才插入 |
| NOT ALL | 不包含全部排除关键词才插入 |

### 插入位置

| 位置 | 说明 |
|------|------|
| ⚙️ 系统 | 作为系统消息插入 |
| 👤 用户 | 作为用户消息插入 |
| 🤖 助手 | 作为助手消息插入 |

### 世界设定激活设置

每个世界设定书的全局设置：

| 设置 | 说明 |
|------|------|
| **Scan Depth** | 扫描最近多少条消息（滑块 0-20） |
| **Context %** | 最大使用多少百分比的上下文 |
| **递归深度** | 允许链式触发的最大层数 |

### 作用域绑定

世界设定可以绑定到：

- **全局**：所有对话都生效
- **角色**：仅在与特定角色对话时生效
- **人设**：仅在特定用户人设激活时生效
- **会话**：仅在特定对话中生效

## 记忆系统

### 手动记忆

1. 进入 **记忆管理** 页面
2. 点击 **添加记忆** 按钮
3. 输入记忆内容、标题、分类、标签

### 自动记忆

启用 **auto-memory** 插件后，系统自动从对话中提取重要信息：

- 包含 "请记住"、"重要的是" 等关键词的内容
- 用户偏好信息（"我最喜欢..."、"我不喜欢..."）
- 身份信息（"我的名字是..."、"我住在..."）
- 随机事实（"顺便一提..."）

### 记忆分类

| 分类 | 说明 |
|------|------|
| `identity` | 身份信息（重要性 0.9） |
| `important` | 重要内容（重要性 0.85） |
| `preference` | 偏好信息（重要性 0.8） |
| `fact` | 事实信息（重要性 0.7） |

### 记忆检索

在对话时，系统根据消息内容自动检索相关记忆，注入到系统提示词中。

## 用户人设 (Personas)

### 什么是人设

人设定义的是 **你（用户）** 的身份，而非 AI 角色。

### 创建人设

1. 进入 **人设** 页面（导航栏 🪪）
2. 点击 **创建人设** 按钮
3. 填写：
   - **名称**：人设名称（如"剑客"、"法师"）
   - **描述**：人设简介
   - **系统提示词**：定义人设的说话方式和行为
   - **头像**：上传用户头像

### 使用人设

创建人设后，在对话中可以选择激活不同的人设，获得不同的对话体验。

### 人设与角色的区别

| | 角色 | 人设 |
|---|---|---|
| 定义对象 | AI 角色 | 用户自己 |
| 影响 | AI 的回复风格 | 上下文中的用户设定 |
| 头像 | 角色图片 | 用户头像 |

## Prompt 管理器

### 功能说明

Prompt 管理器允许你精细控制发送给 AI 的提示词结构，支持可视化拖拽排序。

### 提示词组件

| 组件 | 说明 | 默认状态 |
|------|------|----------|
| ⚙️ 系统提示词 | 全局指令 | 启用 |
| 👤 角色定义 | 当前角色信息 | 启用 |
| 🪪 用户人设 | 当前人设信息 | 启用 |
| 📚 世界设定 | 激活的世界设定条目 | 启用 |
| 🧠 记忆系统 | 相关记忆内容 | 启用 |
| 💬 示例对话 | 对话示例 | 禁用 |
| 📝 Author's Note | 深度提示 | 禁用 |
| ✨ 自定义指令 | 额外指令 | 禁用 |

### 使用方法

1. 进入 **Prompt** 页面（导航栏 📝）
2. 选择要编辑的组件
3. 修改内容
4. 点击 **保存全部**

### 高级功能

- **拖拽排序**：拖动组件左侧 ☰ 手柄调整顺序
- **启用/禁用**：点击开关控制组件是否生效
- **恢复默认**：点击 ↩️ 恢复组件默认值
- **Token 统计**：实时显示每个组件的 token 数量和使用率

### 导入/导出

支持 JSON 格式的配置文件导入导出，方便在不同设备间同步。

## 图片生成

### 支持的生成器

| 生成器 | 说明 |
|--------|------|
| **Stable Diffusion** | 本地或远程 SD 模型（需配置 URL） |
| **DALL-E** | OpenAI 的 DALL-E 服务 |

### 生成参数

| 参数 | 说明 |
|------|------|
| 提示词 | 图片描述 |
| 负面提示词 | 不想出现的元素 |
| 宽度/高度 | 输出尺寸（512/768/1024） |
| 步数 | 生成迭代次数（10-50） |
| CFG 强度 | 提示词遵循程度（1-15） |
| Seed | 随机种子（留空随机） |
| 采样器 | Euler a / DPM++ / DDIM 等 |

### 使用方式

1. 进入 **生图** 页面（导航栏 🖼️）
2. 选择生成器
3. 输入提示词
4. 调整参数
5. 点击生成

### 操作功能

- **下载**：保存生成的图片到本地
- **复制**：复制图片到剪贴板
- **历史记录**：查看最近生成的图片（最多 20 张）

## 常见问题

**Q: AI 不回复怎么办？**

1. 检查 API 配置是否正确
2. 确认 Ollama 服务是否运行（或 API Key 是否有效）
3. 查看后端控制台错误日志

**Q: 回复很慢怎么办？**

1. 使用本地 Ollama 而非远程 API
2. 减少上下文长度
3. 使用更小的模型

**Q: 世界设定不生效？**

1. 检查条目的关键词是否在消息中出现
2. 确认条目的 **触发策略** 是否正确
3. 尝试将 Scan Depth 调高

**Q: 如何导入 SillyTavern 角色卡？**

支持直接导入 SillyTavern JSON 格式的角色卡文件。

**Q: 如何使用 Prompt 管理器？**

在导航栏点击 📝 进入 Prompt 管理器，可以精细控制发送给 AI 的提示词组件，支持拖拽排序、启用/禁用、导入导出配置。

**Q: 图片生成失败？**

1. 确认 Stable Diffusion 服务已启动（或 DALL-E API Key 有效）
2. 检查 SD_API_URL 配置是否正确
