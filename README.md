## 1. 项目简介

AstralLounge 是一个现代化的本地 AI 对话平台，灵感来源于 SillyTavern，提供以下核心能力：

| 功能 | 说明 |
|------|------|
| **多模型支持** | Ollama (本地)、DeepSeek、OpenAI、Anthropic 等 |
| **角色管理** | 创建、编辑、导入、导出 AI 角色 |
| **群聊模式** | 多角色同时参与对话 |
| **世界设定** | 关键词触发的上下文注入 |
| **记忆系统** | 自动 + 手动记忆，支持分类和重要性评分 |
| **插件系统** | 可扩展的技能架构 |
| **语音合成** | CosyVoice / GPT-SoVITS 集成 |
| **深色/浅色主题** | 双语支持，移动端适配 |

### 硬件需求

- **最低配置**: 可运行 Node.js 18+ 和 Python 3.9+ 的设备
- **本地 LLM 推理**: 推荐 NVIDIA 6GB+ 显存显卡（如 RTX 3060）
- **纯 API 模式**: 无需本地显卡

## 2. 安装与启动

### 2.1 快速安装

**步骤 1: 安装后端依赖**

```bash
cd backend
pip install -r requirements.txt
```

**步骤 2: 安装前端依赖**

```bash
cd frontend
npm install
```

**步骤 3: 配置环境变量**

复制环境变量模板：

```bash
cp .env.example .env
```

编辑 `.env` 文件，参考 [第 4 节](#4-ai-模型配置) 配置你的 AI 模型。

**步骤 4: 启动**

- **Windows**: 双击 `start.bat`
- **Linux/macOS**: `./start.sh`

或手动分步启动：

```bash
# 终端 1 - 后端
cd backend
python -m uvicorn backend.main:app --reload --port 8000

# 终端 2 - 前端
cd frontend
npm run dev
```

### 2.2 启动后访问

| 服务 | 地址 |
|------|------|
| 前端界面 | http://localhost:3000 |
| 后端 API | http://localhost:8000 |
| API 文档 | http://localhost:8000/docs |

## 4. AI 模型配置

### 4.1 配置方式

进入 **设置 → 模型**，或直接在 `.env` 文件中配置。

### 4.2 支持的模型类型

#### Ollama (本地模型) — 推荐用于本地推理

```env
DEFAULT_MODEL_TYPE=ollama
OLLAMA_BASE_URL=http://localhost:11434
DEFAULT_MODEL_NAME=llama3.2
```

确保 Ollama 服务已启动：

```bash
ollama serve
ollama list  # 查看已下载的模型
ollama pull llama3.2  # 下载模型
```

#### DeepSeek

```env
DEFAULT_MODEL_TYPE=deepseek
DEEPSEEK_API_KEY=sk-your-api-key
DEEPSEEK_BASE_URL=https://api.deepseek.com/v1
```

#### OpenAI

```env
DEFAULT_MODEL_TYPE=openai
OPENAI_API_KEY=sk-your-api-key
OPENAI_BASE_URL=https://api.openai.com/v1
```

#### Anthropic (Claude)

```env
DEFAULT_MODEL_TYPE=anthropic
ANTHROPIC_API_KEY=sk-ant-your-api-key
```

### 4.3 模型参数

在设置页面可以调整：

| 参数 | 说明 | 推荐范围 |
|------|------|----------|
| Temperature | 创造性/确定性 | 0.5 ~ 1.0 |
| Max Tokens | 单次回复最大长度 | 1024 ~ 8192 |

## 5. 角色管理

### 5.1 创建角色

1. 点击 **🎭 角色** 进入角色管理页面
2. 点击 **+ 创建** 按钮
3. 填写角色信息：

| 字段 | 说明 |
|------|------|
| 名称 | 角色名字 |
| 描述 | 角色的简要介绍 |
| 性格 | 描述角色的性格特征 |
| 场景 | 对话发生的背景设定 |
| 开场白 | 角色说的第一句话 |
| 示例对话 | 提供对话示例 (User: / Character:) |
| 标签 | 用于分类和搜索 |
| 头像 | 上传角色图片 |

### 5.2 角色字段详解

**性格 (Personality)**

这是最重要的字段之一。详细描述角色的：
- 性格特征（外向/内向、温柔/强势等）
- 说话风格（正式/口语化、爱用某些口头禅等）
- 情感倾向（乐观/悲观、理性/感性等）

示例：

```
温柔体贴，善于倾听，说话轻声细语。
喜欢用"呢""呀"等语气词。
对陌生人有些害羞，熟悉后会变得话多。
遇到困难时会先安慰对方再想办法。
```

**场景 (Scenario)**

设定对话发生的背景环境：

```
咖啡厅的角落里，下午的阳光透过窗户洒进来。
两人是同事关系，正在讨论工作之余的话题。
```

**开场白 (Greeting)**

角色主动发起的开场白，会在新建对话时自动发送。

### 5.3 导入角色

支持从以下格式导入角色卡：

- **SillyTavern 格式** (`.json`)
- **TavernAI 格式** (`.json`)
- **Ooba 格式** (`.json`)
- **图片嵌入格式** (`.png` / `.webp`)

**导入方式**：

1. 点击 **导入** 按钮
2. 选择 **文件导入**（拖拽或点击选择）或 **粘贴导入**
3. 支持的文件格式：`JSON`、`PNG`、`JPG`、`WEBP`

### 5.4 导出角色

1. 在角色卡片上点击 **📤** 导出按钮
2. 选择导出格式：`SillyTavern`、`TavernAI`、`Ooba`、`JSON`
3. 点击下载

## 6. 对话功能

### 6.1 基本对话

1. 进入 **💬 对话** 页面
2. 点击 **+ 新对话** 创建会话
3. 在输入框输入消息，按 `Enter` 发送
4. AI 回复以流式方式逐字显示

### 6.2 关联角色

1. 在输入框上方点击 **🎭** 按钮
2. 从角色列表中选择一个角色
3. 角色信息（性格、场景）会作为系统提示注入对话

### 6.3 Ghost 防护模式

Ghost 模式是一种消息确认机制，防止 AI 产生意外输出：

1. 在对话头部点击 **👻** 按钮开启 Ghost 模式
2. 发送消息后，AI 回复会先显示在确认条中
3. 你有 **5 秒**时间决定：
   - **确认**: 将回复加入对话
   - **放弃**: 丢弃回复
4. 超时自动放弃

### 6.4 会话管理

| 操作 | 说明 |
|------|------|
| 置顶 | 点击 📌 固定会话到列表顶部 |
| 重命名 | 点击 ✏️ 修改会话名称 |
| 删除 | 点击 🗑️ 删除会话 |
| 导出 | 点击 📤 导出会话为 JSON |
| 分组 | 使用分组整理大量会话 |

### 6.5 分组管理

1. 点击侧边栏的 **📁** 按钮显示分组栏
2. 点击 **+ 创建分组** 设定名称和颜色
3. 在会话列表按分组筛选

## 8. 世界设定 (Lorebook)

世界设定允许你在对话中动态注入上下文信息，基于关键词自动触发。

### 8.1 创建世界设定

1. 进入 **📚 世界** 页面
2. 点击 **创建世界设定**
3. 填写名称和描述

### 8.2 添加条目

每个世界设定由多个条目组成，每个条目包含：

| 字段 | 说明 |
|------|------|
| 名称 | 条目标题 |
| 内容 | 要注入的上下文内容 |
| 关键词 | 触发该条目的关键词（逗号分隔） |
| 优先级 | 数值越大优先级越高 |
| 启用 | 是否激活该条目 |

**关键词机制**：

当对话中检测到关键词时，对应条目内容会自动插入上下文。例如：

```
条目名称: 城市设定
关键词: 首都, 王城, 王宫
内容: 这是一个位于大陆中央的繁华首都，王宫坐落在北部的山丘上。
```

当用户提到"首都""王城"或"王宫"时，这段内容会被注入。

### 8.3 条目配置参数

| 参数 | 说明 |
|------|------|
| Scan Depth | 向前扫描多少条消息检测关键词 |
| Context Length | 最大注入 token 数 |
| Insert Mode | `append`（追加）或 `insert`（插入） |
| Force Activation | 强制激活，不依赖关键词 |
| Enabled | 整体启用/禁用 |

## 9. 记忆系统

AstralLounge 提供两种记忆模式：

### 9.1 手动记忆

1. 进入 **🧠 记忆** 页面
2. 点击 **创建记忆**
3. 填写内容：

| 字段 | 说明 |
|------|------|
| 标题 | 记忆标题 |
| 内容 | 记忆具体内容 |
| 分类 | 如"人物关系""世界观""剧情" |
| 标签 | 额外的分类标签 |
| 重要性 | 1-10 分，影响记忆检索优先级 |
| 来源 | `manual`（手动）或 `auto`（自动） |

### 9.2 自动记忆

插件系统可以自动从对话中提取重要信息存入记忆。参考 [第 10 节](#10-插件系统) 中的 `auto_memory` 插件。

### 9.3 搜索记忆

1. 使用搜索框按内容关键词搜索
2. 按分类或标签筛选
3. 高访问频率的记忆会获得更高检索权重

## 10. 插件系统

### 10.1 内置插件

#### auto_memory (自动记忆)

自动从对话中提取重要信息存入记忆系统。

在 `backend/plugins/skills/auto_memory.py` 中配置触发关键词。

#### web_search (网络搜索)

允许 AI 实时搜索网络信息。

### 10.2 插件生命周期

| 方法 | 时机 |
|------|------|
| `on_load()` | 插件加载时 |
| `on_unload()` | 插件卸载时 |
| `process_input()` | 处理用户输入 |
| `process_output()` | 处理 AI 输出 |
| `on_message()` | 拦截消息 |

### 10.3 插件管理

进入 **🔌 插件** 页面可以：

- 查看已加载的插件列表
- 查看插件状态和触发词
- 重载单个插件或全部插件

### 10.4 开发自定义插件

在 `backend/plugins/skills/` 目录下创建 Python 文件：

```python
from backend.plugins.plugin_manager import BaseSkill, SkillMetadata

class MySkill(BaseSkill):
    metadata = SkillMetadata(
        name="my-skill",
        version="1.0.0",
        description="我的自定义技能",
        author="Your Name",
        triggers=["触发词1", "触发词2"],
        enabled=True,
        category="tools"
    )

    async def on_load(self):
        print("插件加载成功")

    async def process_input(self, text: str, context: dict) -> str:
        # 处理输入消息
        return text

    async def process_output(self, text: str, context: dict) -> str:
        # 处理输出消息
        return text
```

## 11. 语音合成 (TTS)

### 11.1 启用 TTS

1. 进入 **⚙️ 设置 → TTS**
2. 开启 **启用语音合成**
3. 选择提供商：

| 提供商 | 说明 | 地址 |
|--------|------|------|
| CosyVoice | 阿里巴巴开源 TTS | http://localhost:5000 |
| GPT-SoVITS | 开源语音合成 | http://localhost:5001 |

### 11.2 音色设置

支持以下音色：

| 音色 ID | 名称 |
|---------|------|
| female_zh | 女声-中文 |
| male_zh | 男声-中文 |
| female_en | 女声-英文 |
| male_en | 男声-英文 |

### 11.3 语速调节

拖动滑块调整语速，范围 **0.5x ~ 2.0x**。

### 11.4 使用方式

在对话页面的输入工具栏中：

1. 点击 **🔊** 打开 TTS 面板
2. 输入要朗读的文本
3. 选择音色和语速
4. 点击 **播放** 按钮

## 14. 进阶技巧

### 14.1 提升对话质量

1. **详细编写角色性格**: 越详细的性格描述，AI 越能保持角色一致性
2. **使用开场白**: 好的开场白能快速建立对话氛围
3. **提供示例对话**: 帮助 AI 理解期望的对话风格
4. **合理使用世界设定**: 不要一次注入过多内容，避免上下文稀释

### 14.2 优化 Token 使用

- 开启 **上下文 Token 计数警告**
- Token 超过 5000 显示警告，超过 8000 显示危机警告
- 适时开启 **新对话** 清理上下文

### 14.3 Ghost 模式工作流

推荐用于：
- 生成敏感内容时二次确认
- 调整 AI 行为时预览效果
- 长时间等待生成结果时先处理其他事务

### 14.4 多角色对话技巧

在群聊中：
- 设置成员顺序决定发言轮换
- 每个角色的开场白决定群聊氛围
- 使用世界设定注入共同背景信息

### 14.5 与 SillyTavern 角色卡兼容

AstralLounge 支持导入 SillyTavern 的角色卡格式（JSON 和图片嵌入 PNG），方便从 SillyTavern 迁移数据。

## 15. 常见问题

### Q: 启动报错 "Module not found"

确保已安装所有依赖：

```bash
pip install -r backend/requirements.txt
cd frontend && npm install
```

### Q: 无法连接到 Ollama

确认 Ollama 服务正在运行：

```bash
ollama serve
ollama list  # 查看已下载的模型
```

### Q: DeepSeek API 调用失败

1. 检查 API Key 是否正确配置
2. 确认网络可以访问 `api.deepseek.com`
3. 查看后端日志 `data/logs/` 获取详细错误

### Q: 前端样式异常

清除浏览器缓存，或在设置中切换主题后再切回。

### Q: 语音合成无声音

1. 确认 TTS 服务（CosyVoice/GPT-SoVITS）已启动
2. 检查 `TTS_ENABLED=true` 和 `TTS_PROVIDER` 配置
3. 确认 TTS 服务地址可访问

### Q: 导入角色失败

- JSON 格式不兼容：尝试复制 JSON 内容使用 **粘贴导入**
- 图片嵌入角色卡：确保图片为 `.png` 或 `.webp` 格式
- 查看后端日志获取具体错误信息

### Q: 如何查看 API 文档？

访问 http://localhost:8000/docs 查看完整的 Swagger API 文档（调试模式）。
